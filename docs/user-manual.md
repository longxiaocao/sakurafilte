# SakuraFilter 用户手册

> 面向最终用户的操作指南。
> 涵盖访客（无需登录）的公开功能、管理员/操作员/查看者三种角色后台功能。
> 技术细节参见 [`ops-manual.md`](./ops-manual.md) / [`api-contract.md`](./api-contract.md)。

## 1. 系统简介

SakuraFilter 是一个面向汽配行业的**产品/交叉引用/机型应用**数据库，提供：

- **OEM 替代查询**：输入原厂件号，找到兼容的替代品
- **机型适配查询**：根据车型/发动机/机型，匹配适用配件
- **产品对比**：把多个产品放在一起横向比较
- **数据管理后台**：导入产品数据、维护字典、查看监控

## 2. 访客功能（无需登录）

访客（未登录用户）可使用以下功能：

### 2.1 搜索产品

**入口**：`/` 或 `/search`

| 搜索字段 | 说明 | 示例 |
|---|---|---|
| OEM 件号 | 完整或部分件号 | `AC 0101` |
| OEM 品牌 | 品牌名 | `BOSCH` |
| 产品名 1 | 主产品类型 | `Oil Filter` |
| 产品名 2 | 副产品类型 | `Spin-on` |
| 机型品牌 | 适用机型品牌 | `CATERPILLAR` |
| 机型代号 | 适用机型代号 | `320D` |
| 机型类别 | 适用机型类别 | `EXCAVATOR` |
| 媒介 | 油/水/气等 | `OIL` |

**容差控制**（尺寸类搜索）：右上角"容差"下拉框，可选 `±0/±1/±5/±10mm`。

**操作技巧**：
- 任一字段命中即返回（OR 关系）
- 字段留空 = 不过滤该字段
- 关键词越具体，结果越精确
- 翻页时使用"上一页/下一页"按钮，URL 自动携带 HMAC 签名 cursor

### 2.2 查看产品详情

**入口**：搜索结果中点击任一产品 / 直接访问 `/product/<OEM编号>`

页面包含：
- 基础信息：OEM 编号、品牌、类型、产品名称、尺寸
- 适用机型：哪些机型/发动机使用该产品
- 交叉引用：其他可替代的 OEM 件号
- 产品图片（如有）
- 操作按钮：
  - **查询替代**：跳转到当前页面的替代品表格（带数量提示，例如 "查询替代 (14)"）
  - **加入对比**：添加到对比列表（带产品 ID 上下文，跳转后无需手动输入）

### 2.3 产品对比

**入口**：`/compare?ids=1,2,3`（URL 带产品 ID，刷新不丢上下文）

- **匿名访客**：最多对比 5 个产品
- **登录用户**：最多对比 20 个产品
- 对比页面以表格形式展示：
  - 行：每个产品
  - 列：OEM、品牌、类型、尺寸、机型、交叉引用数量
  - 差异高亮：相同字段灰底，差异字段黄底

### 2.4 全局拖拽上传（2026-07 新增）

将 `.xlsx` / `.jsonl` 文件**拖到页面任意位置**即可触发上传：
- 系统自动识别文件类型
- `.xlsx` 进入 ETL 导入队列（需登录）
- `.jsonl` 自动按内容识别实体类型
- 拖拽时显示半透明遮罩 + "松手上传"提示

## 3. 登录与角色

### 3.1 登录

**入口**：`/login`

```text
用户名: <您的账号>
密码:   <您的密码>
```

登录成功后跳转到原访问页面（带 returnUrl），不会丢失上下文。

### 3.2 角色说明

| 角色 | 权限范围 |
|---|---|
| **admin**（管理员） | 全部功能，含用户管理、敏感配置 |
| **operator**（操作员） | ETL 导入、字典维护、产品增删改、查看监控 |
| **viewer**（查看者） | 只读浏览后台、查看监控、查看审计日志 |

### 3.3 修改密码

**入口**：右上角头像 → 修改密码

- 旧密码验证
- 新密码要求：8-128 字符，至少 1 字母 + 1 数字
- 修改成功后需重新登录

### 3.4 退出登录

**入口**：右上角头像 → 退出

- 后端撤销当前 refresh token
- 前端清除本地 access token

## 4. 管理员功能

### 4.1 产品管理

**入口**：`/admin/products`

#### 4.1.1 列表

- 默认每页 20 条，可切换 50/100
- 列：OEM、品牌、类型、产品名、尺寸、机型数、引用数、状态、操作
- 顶部搜索框：按 OEM 编号 / 品牌 / 名称模糊搜索（回车触发）
- 高级筛选：点击"显示所有列"展开全部字段
- 批量操作：勾选多条后批量启用/禁用/删除

#### 4.1.2 新增产品

点击"新增产品"按钮：
1. 填写基础字段（OEM、品牌、类型、名称）
2. 可选：上传产品图片（拖拽到图片区）
3. 提交后立即入库，刷新可看

#### 4.1.3 编辑产品

点击行末"编辑"按钮：
- 字段修改即时校验
- 乐观锁：并发编辑时后保存者提示"已被他人修改"
- 撤销修改：右上角"取消"按钮

#### 4.1.4 停用/启用

- 停用产品在公开搜索结果中**不展示**，但历史数据保留
- 适用于下架、停产等场景

### 4.2 字典管理

系统提供 7 个维度的字典维护：

| 字典 | 路径 | 说明 |
|---|---|---|
| 配件类型（Type） | `/admin/dict/type` | 一级分类，如滤清器/刹车片 |
| 配件名称 1（ProductName1） | `/admin/dict/product-name1` | 主类型名称 |
| 配件名称 2（ProductName2） | `/admin/dict/product-name2` | 副类型名称 |
| 机型类别（MachineCategory） | `/admin/dict/machine` | 挖掘机/装载机等 |
| 发动机品牌（EngineBrand） | `/admin/dict/engine` | CUMMINS / ISUZU 等 |
| 媒介（Media） | `/admin/dict/media` | OIL / WATER / AIR |
| OEM 品牌（OemBrand） | `/admin/dict/oem-brands` | 品牌简称/全称映射 |
| OEM 件号 3（OemNo3） | `/admin/dict/oem-no3` | 第三方件号归一化 |

每个字典页面：
- 列表：编码、名称、排序、状态
- 新增：编码自动小写，名称可中文
- 编辑：双击行内任意字段
- 删除：软删除（保留历史关联）

### 4.3 ETL 数据导入

**入口**：`/admin/etl`

#### 4.3.1 支持的文件格式

| 格式 | 说明 |
|---|---|
| `.xlsx` | Excel 文件，按 sheet 识别实体类型 |
| `.jsonl` | 每行一条 JSON 记录，文件名带实体类型 |

#### 4.3.2 触发方式

**方式 1：拖拽上传**
- 把文件拖到页面任意位置
- 自动识别类型并入队

**方式 2：手动上传**
- 点击"选择文件"按钮
- 选择模式：`upsert`（默认）/ `insertonly` / `fullload`
- 提交后立即触发

#### 4.3.3 实时监控

- 顶部进度条：当前处理百分比
- 实时计数：已处理/总数/错误数
- SSE 实时推送：跨实例广播（其他管理员也能看到）
- 暂停/继续：紧急情况下可暂停任务
- 取消：带原因码（USER_REQUEST / DATA_ERROR / SYSTEM）

#### 4.3.4 任务历史

- 最近 20 次任务的执行情况
- 含耗时、错误数、文件路径、触发人

#### 4.3.5 死信队列

- 错误记录自动入队，可手动重放
- 配置自动清理周期（默认 7 天）

### 4.4 性能监控

**入口**：`/admin/perf`

| 指标 | 说明 |
|---|---|
| P50 / P95 / P99 | 请求延迟分位数 |
| 错误率 | 5xx 占比 |
| 最大延迟 | 单次最慢请求 |
| 总请求数 | 累计计数 |
| 样本数 | 环形缓冲区长度（最近 1000 条） |

**告警**：超过阈值自动 webhook 通知（需配置）。

### 4.5 用户管理（仅 admin）

**入口**：`/admin/users`

| 操作 | 说明 |
|---|---|
| 列表 | 用户名、邮箱、角色、状态、最近登录 |
| 新建 | 用户名（字母数字下划线连字符）、密码、角色、邮箱 |
| 编辑 | 修改角色 / 邮箱 / 姓名 / 启用状态 |
| 重置密码 | 管理员直接重置，无需旧密码 |
| 软删除 | 停用账户（保留审计日志） |
| 登录审计 | 查 `login_audit_logs` 表分页 |

### 4.6 主题切换

**入口**：右上角"主题"按钮

- 浅色 / 深色 / 跟随系统
- 偏好保存到 `localStorage`，跨设备不同步（基于本地存储）
- 切换无刷新、即时生效

## 5. 常见问题（FAQ）

### Q1: 搜索不到某个产品？

**可能原因**：
- 产品已被停用（在 `/admin/products` 中 `is_active=false`）
- OEM 编号拼写错误
- 容差设置过窄

**排查**：
1. 切换容差为 `±10mm`
2. 只填 OEM 编号（其他字段清空）
3. 联系管理员确认产品状态

### Q2: 替代品太多，怎么筛选？

**技巧**：
- 加上"产品名 1"或"产品名 2"过滤
- 加上"机型"过滤
- 在结果页用浏览器 `Ctrl+F` 二次搜索

### Q3: 拖拽上传没反应？

**排查**：
1. 浏览器是否支持 HTML5 拖拽 API（IE 不支持）
2. 文件大小 > 100MB 时可能超时，建议拆分
3. 检查浏览器控制台是否有报错
4. 文件类型必须是 `.xlsx` / `.jsonl`

### Q4: 登录后页面空白？

**原因**：JWT token 过期（默认 30 分钟）。

**处理**：
- 刷新页面会自动跳转到登录
- 登录成功后回到原页面

### Q5: 管理员密码忘了？

**处理**：
- 找另一个 admin 角色账号在 `/admin/users` 重置
- 都没人记得时需要数据库直接修改：
  ```sql
  UPDATE users SET password_hash = '<bcrypt of new password>' WHERE username = 'admin';
  ```
  （bcrypt 哈希由后端 `UserService.CreateAsync` 生成，参考代码实现）

### Q6: 导入任务卡住不动？

**排查**：
1. 看进度条和实时计数：是否在 0% 停住（可能还没启动）或 50% 停住（可能某行数据异常）
2. 看后端日志：`grep "ETL" /var/log/sakurafilter/api.log | tail -50`
3. 死信队列是否堆积：进 `/admin/etl` → "死信队列"标签
4. 必要时点击"取消"，再重新上传

### Q7: 公开访问是否需要登录？

**不需要**。以下路径**匿名可访问**：

- `/`（首页）
- `/search`（搜索）
- `/product/:oem`（产品详情）
- `/compare?ids=...`（对比）
- `/api/public/*`（公开 API）

只有 `/admin/*` 路径需要登录。

## 6. 术语表

| 术语 | 含义 |
|---|---|
| **OEM** | Original Equipment Manufacturer，原厂件号 |
| **交叉引用** | Cross Reference，可替代的 OEM 件号 |
| **机型应用** | Machine Application，适用该配件的车型/发动机 |
| **ETL** | Extract-Transform-Load，数据导入流程 |
| **死信队列** | Dead Letter Queue，错误数据暂存区 |
| **乐观锁** | Optimistic Locking，并发编辑冲突检测 |
| **HMAC** | Hash-based Message Authentication Code，cursor 防篡改 |
| **SSE** | Server-Sent Events，服务端实时推送 |
| **PG NOTIFY/LISTEN** | PostgreSQL 跨实例消息广播机制 |
| **熔断器** | Circuit Breaker，外部依赖故障时自动降级 |
